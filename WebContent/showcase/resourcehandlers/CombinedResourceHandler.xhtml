This ResourceHandler implementation will remove all separate script and stylesheet resources which have the target attribute set to "head" from the UIViewRoot and create a combined one for all scripts and another combined one for all stylesheets.

This handler must be registered as follows in faces-config.xml in order to get it to run: 


<application>
    <resource-handler>org.omnifaces.resourcehandler.CombinedResourceHandler</resource-handler>
</application>

Note that the target attribute of <h:outputStylesheet> already defaults to "head" but the one of <h:outputScript> not. So if you have placed this inside the <h:head>, then you would still need to explicitly set its target attribute to "head", otherwise it will be treated as an inline script and not be combined. This is a design limitation. 


<h:head>
    <h:outputStylesheet name="style.css" />
    <h:outputScript name="script.js" target="head" />
</h:head>

If you want them to appear after any auto-included resources of standard JSF implementation or JSF component libraries, then move the declarations to the <h:body>. 


<h:body>
    <h:outputStylesheet name="style.css" />
    <h:outputScript name="script.js" target="head" />
</h:body>

The CSS one has combined the three CSS files primefaces.css, layout.css and prettify.css. The JS one has combined the five JS files jquery.js, primefaces.js, jsf.js, omnifaces.js and prettify.js. Note that the PrimeFaces theme.css file cannot be combined because it's not been added as a JSF resource, but hardcoded in PrimeFaces' HeadRenderer so that it could be generated as the very first CSS link to avoid CSS inheritance problems in custom CSS files which are supposed to override PrimeFaces default CSS. There's however a way to get it to be included anyway, but this requires a custom renderer for the <head> wherein the theme name is been hardcoded as @ResourceDependency. See also How to combine PrimeFaces theme.css using CombinedResourceHandler anyway?

The generated combined resource URL also includes the "v" request parameter which is the last modified time of the newest individual resource in minutes, so that the browser will always be forced to request the latest version whenever one of the individual resources has changed.
